Clases Abstractas e Interfaces

Una guía sencilla con ejemplos en Java y Python.

1. Clases Abstractas

Una clase abstracta es una "plantilla" para otras clases. No puedes crear un objeto directamente de ella, pero otras clases pueden heredarla. Es útil cuando quieres proporcionar una funcionalidad base que las clases hijas compartirán y obligarlas a implementar ciertos métodos.

Inicio

Java

// No se puede instanciar: new Figura() da error.
abstract class Figura {
    // Atributo que heredan las clases hijas.
    String color;

    // Método abstracto: sin implementación.
    // Las clases hijas ESTÁN OBLIGADAS a implementarlo.
    abstract double area();

    // Método concreto: con implementación.
    // Las clases hijas lo heredan tal cual.
    public void mostrarColor() {
        System.out.println("Color: " + color);
    }
}

class Circulo extends Figura {
    double radio;

    // Implementación obligatoria del método abstracto.
    @Override
    public double area() {
        return Math.PI * radio * radio;
    }
}

Python

# Python usa el módulo 'abc' (Abstract Base Classes)
from abc import ABC, abstractmethod

# Hereda de ABC para ser abstracta.
class Figura(ABC):
    def __init__(self, color):
        # Atributo que heredan las clases hijas.
        self.color = color

    # Decorador para marcar el método como abstracto.
    @abstractmethod
    def area(self):
        pass

    # Método concreto que heredan las hijas.
    def mostrar_color(self):
        print(f"Color: {self.color}")


class Circulo(Figura):
    def __init__(self, color, radio):
        super().__init__(color)
        self.radio = radio

    # Implementación obligatoria del método abstracto.
    def area(self):
        import math
        return math.pi * self.radio ** 2

2. Interfaces

Una interfaz es un "contrato" que una clase puede firmar. Define un conjunto de métodos que la clase se compromete a implementar. Una interfaz solo dice QUÉ debe hacer una clase, pero no CÓMO. Son la base del polimorfismo y permiten que objetos muy diferentes compartan una misma capacidad.

Java

// Una interfaz es un contrato 100% abstracto.
interface Volador {
    // Todos los métodos son públicos y abstractos por defecto.
    void despegar();
    void aterrizar();
}

class Avion implements Volador {
    // Se obliga a implementar TODOS los métodos de la interfaz.
    @Override
    public void despegar() {
        System.out.println("Avión despegando...");
    }
    @Override
    public void aterrizar() {
        System.out.println("Avión aterrizando...");
    }
}

class Pajaro implements Volador {
    @Override
    public void despegar() {
        System.out.println("Pájaro batiendo alas para despegar.");
    }
    @Override
    public void aterrizar() {
        System.out.println("Pájaro planeando para aterrizar.");
    }
}

Python

# Python no tiene 'interfaces' como tal. Se logra el
# mismo objetivo con Clases Base Abstractas (ABC).
# La idea es crear un contrato 100% abstracto.
from abc import ABC, abstractmethod

class Volador(ABC):
    @abstractmethod
    def despegar(self):
        pass

    @abstractmethod
    def aterrizar(self):
        pass

class Avion(Volador):
    # Se obliga a implementar TODOS los métodos abstractos.
    def despegar(self):
        print("Avión despegando...")

    def aterrizar(self):
        print("Avión aterrizando...")

class Pajaro(Volador):
    def despegar(self):
        print("Pájaro batiendo alas para despegar.")
    
    def aterrizar(self):
        print("Pájaro planeando para aterrizar.")

Diferencias Clave: Resumen Rápido

Clase Abstracta

  • Puede tener métodos abstractos y concretos (con código).
  • Puede tener atributos/propiedades de instancia.
  • Una clase solo puede heredar de UNA clase abstracta (herencia simple).
  • Define una relación de tipo "ES UN" (un Círculo es una Figura).

Interfaz

  • Generalmente, solo define métodos abstractos (el contrato).
  • No puede tener atributos de instancia (solo constantes).
  • Una clase puede implementar MÚLTIPLES interfaces.
  • Define una relación de tipo "PUEDE HACER" (un Avión puede volar).

¿Cuándo usar cada uno?

Usa una Clase Abstracta si:

Quieres compartir código (métodos concretos) o estado (atributos) entre varias clases que están estrechamente relacionadas. Por ejemplo, `Gato` y `Perro` podrían heredar de una clase abstracta `Animal` que ya implementa el método `dormir()`.

Usa una Interfaz si:

Necesitas que clases muy diferentes (y no relacionadas) tengan la misma capacidad o comportamiento. Por ejemplo, `Avion`, `Pajaro` y `Superman` podrían implementar la interfaz `Volador`, aunque no tengan nada más en común.

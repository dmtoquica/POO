Proyecto 1: Sistema de Simulación y Control para Atracción Mecánica

Colaborativo con Arquitectura de Hardware

Necesidad a Resolver

Se requiere un software central que actúe como un "gemelo digital", no solo simulando el comportamiento de una atracción, sino también controlando directamente el prototipo físico. El software debe enviar comandos que activen componentes mecánicos reales y su simulación gráfica debe reflejar el estado exacto del hardware en tiempo real.

Entregables

  • ➔Diagrama de Clases UML: Diseño detallado de la arquitectura del software.
  • ➔Código Fuente Completo: Implementación robusta aplicando todos los conceptos de POO.
  • ➔GUI Funcional: Panel de control visual para monitoreo y operación en tiempo real.
  • ➔Módulo de Control Funcional: Capa de comunicación (API, Sockets, etc.) para interactuar con el hardware.
  • ➔Video de Demostración Final: Prueba de la interacción exitosa entre software y hardware.

Proyecto 2: CineManager - Sistema de Gestión Integral para Cines

Aplicación de Escritorio

Necesidad a Resolver

Los cines modernos necesitan un sistema digital centralizado para gestionar su operación: venta de boletos, administración de cartelera, gestión de salas y control de ventas de cafetería. El objetivo es optimizar la logística y mejorar la experiencia del cliente a través de una única plataforma.

Entregables

  • ➔Diagrama de Clases UML: Modelado de las entidades del cine y sus relaciones.
  • ➔Código Fuente Completo: Desarrollo de toda la lógica de negocio.
  • ➔GUI Funcional: Aplicación de escritorio con perfiles de Cliente y Administrador.
  • ➔Persistencia de Datos: Simulación de base de datos usando archivos (JSON, XML).

Proyecto 3: StratoServe - Simulador de Arquitectura en la Nube

Enfoque en Patrones de Diseño

Necesidad a Resolver

Para entender servicios como Google Drive, es necesario modelar su arquitectura de software. Se requiere un simulador que replique la lógica de gestión de usuarios, la estructura jerárquica de archivos/carpetas y los planes de suscripción con cuotas, enfocándose en la arquitectura y no en el almacenamiento físico de archivos.

Entregables

  • ➔Diagrama de Clases UML: Diseño que evidencie el uso de patrones de diseño (ej. Composite, Strategy).
  • ➔Código Fuente Completo: Implementación del simulador.
  • ➔GUI Funcional: Interfaz para interactuar con el sistema de archivos virtual.
  • ➔Documentación Técnica: Explicación de la implementación de los patrones de diseño.

Proyecto 4: TurnQuest - Motor para Juego de Rol (RPG) por Turnos

Enfoque en Lógica de Juego

Necesidad a Resolver

El corazón de un RPG es su motor, que gestiona reglas, estadísticas y habilidades. Se necesita construir el núcleo lógico de un juego, creando una arquitectura POO flexible y extensible para añadir fácilmente nuevos personajes, enemigos y objetos en el futuro, enfocándose en la reusabilidad y el polimorfismo.

Entregables

  • ➔Diagrama de Clases UML: Diseño con jerarquías de herencia claras.
  • ➔Código Fuente Completo: Implementación de todo el motor de juego.
  • ➔GUI Jugable y Funcional: Interfaz visual para crear un personaje y participar en un combate.
  • ➔Pruebas Unitarias: Pruebas que demuestren el correcto funcionamiento de la lógica clave.
